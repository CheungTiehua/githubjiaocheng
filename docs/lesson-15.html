二、GitHub 快捷键

在 GitHub 任意页面按 ? 可以查看全部快捷键。

仓库页面

快捷键 作用
. 打开 VS Code 网页版编辑器
t 快速跳转文件（打开文件搜索）
s 或 / 聚焦搜索框
l 跳转到某一行（在文件查看页）
w 切换分支/标签选择
g c 去 Code 标签
g i 去 Issues 标签
g p 去 Pull requests 标签
g a 去 Actions 标签
g w 去 Wiki 标签

Issue / PR 页面

快捷键 作用
c 创建新 Issue（在 Issue 列表页）
r 引用回复（在光标在评论框时）
e 编辑当前 Issue/PR 标题或正文
Ctrl + Enter 提交评论

全站

快捷键 作用
? 显示所有快捷键
Esc 关闭当前弹窗/取消聚焦

三、推荐设置清单

这是 GitHub 最佳实践配置，适用于个人和团队项目。

个人账号

设置 路径 推荐
个人资料 Settings → Public profile 填 Name、Bio、Location
暗色主题 Settings → Appearance Dark mode
SSH Key Settings → SSH and GPG keys 添加至少一个
Token Settings → Developer settings 生成 PAT，过期时间设一年
2FA Settings → Password and authentication 开启！

仓库设置（团队/开源项目）

设置 路径 推荐值
默认分支名 Settings → General main
合并方式 Settings → General 允许 Squash and merge
合并后自动删分支 Settings → General ✅ 开启
分支保护 Settings → Branches main 分支必须 PR + 至少 1 人 Approve
Issue 模板 .github/ISSUE_TEMPLATE/ Bug Report + Feature Request
PR 模板 .github/PULL_REQUEST_TEMPLATE.md 要求填 Summary
CODEOWNERS .github/CODEOWNERS 核心目录设 Owner
Discussions Settings → General → Features 开启
CI .github/workflows/ci.yml 至少跑测试

四、"我要做 X，该点哪里？"速查

我要 操作路径
下载项目代码 仓库主页 → Code → Download ZIP
给开源项目贡献代码 Fork → Clone → 改代码 → Push → Compare & PR
报告一个 Bug Issues → New issue → 描述 → Submit
看某行代码谁写的 文件 → Blame
看某个版本发布的代码 仓库页 → Tags → 选版本号
查某个命令怎么用 全站搜索框 → 输入关键词
同步我的 Fork 自己 Fork 页 → Fetch upstream → Fetch and merge
添加协作者 Settings → Collaborators → Add people
设置 main 分支保护 Settings → Branches → Add rule → 输入 main
看 CI 为什么挂了 Actions → 点失败的 Run → 展开失败 Step → 看红色报错
把自己的代码给别人审核 创建 PR → 右侧 Reviewers → 选人
审核别人的代码 PR → Files changed → 行上加评论 → Review changes → Approve
